# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A GitHub Action (TypeScript) that deletes merged PR head branches, with glob-based inclusion/exclusion rules and optional retargeting of open PRs that use the deleted branch as their base.

## Commands

- `npm run build` — compile TypeScript (`src/` → `dist/`) via `tsc`.
- `npm run package` — run build, then bundle with `@vercel/ncc` into `dist/index.js` with source maps. Produces the single-file artefact the Action runs.
- `npm test` — run Jest (ts-jest preset).
- `npx jest src/util.spec.ts -t "example-branch"` — run a single test by name.

## Architecture

Two source files do all the work:

- `src/index.ts` — the Action entry point invoked at runtime. Reads `GITHUB_TOKEN` (or `GH_PAT`, or the `token` input), pulls the PR from `context.payload.pull_request`, and:
  1. Bails early if another open PR still has this branch as its head.
  2. Calls `shouldDelete` to check the branch name against the `branches` glob rules.
  3. If `retarget_base` is set, lists open PRs with base=<this branch>, moves them onto the retarget base, then re-lists to make sure none remain before deleting.
  4. Calls `git.deleteRef` on `heads/<branch>`.
  - Uses `((octokit as any).rest ?? octokit)` for compatibility across `@actions/github` versions that moved REST calls behind `.rest`.
- `src/util.ts` — `shouldDelete(branch, rules)`. Rules is a comma-separated list of globs; the branch must match **every** rule (`.every(...)`), which is how `!master,!main,*` works as a default: the branch must be neither `master` nor `main` AND match `*`. Changing this to `.some` would silently break exclusion semantics.

## Build artefact is committed

`dist/index.js` is the file the Action executes (see `action.yml`: `runs.main: dist/index.js`, `using: node20`). `.github/workflows/build.yml` runs `npm run package` on pushes to `master` and auto-commits the rebuilt `dist/` via `stefanzweifel/git-auto-commit-action`. When editing `src/`, the committed `dist/` will be regenerated by CI, but be aware that PRs merging from forks may show stale `dist/` until the post-merge build runs. Never hand-edit `dist/`.

## Release flow

`semantic-release` with the `semantic-release-gitmoji` plugin (`.releaserc.json`). Gitmoji prefixes `:bento:` and `:recycle:` are mapped to patch releases; other mappings are plugin defaults. Commits produced by CI use `:rocket: Deploy new version [skip ci]` and `:bookmark: v<version> [skip ci]` — the `[skip ci]` guard is what prevents build loops, and the workflows also filter on head commit messages (`Build CI` skips `[skip ci]`; `Test CI` only runs on `Deploy new version` commits).

## Action inputs (action.yml)

- `branches` — glob rules, default `*,!master,!main`. Note: README documents the default as `!master, !develop`, but `action.yml` is the source of truth.
- `retarget_base` — optional branch name; when set, open PRs targeting the deleted branch are moved here before the ref is deleted.
